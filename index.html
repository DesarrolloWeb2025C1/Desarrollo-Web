HTML y CSS

Introducción a HTML y CSS 👶🏻

Ver más

HTML (HyperText Markup Language) es un lenguaje de marcado que nos permite crear páginas web mediante etiquetas o elementos. Mediante HTML podemos incluir dentro de nuestras páginas web texto, imágenes, enlaces, tablas, vídeos, música, cualquier elemento que podamos imaginar. Todo elemento es representado por una etiqueta.

  • Las etiquetas HTML suelen venir en pares, como <p> ... </p>, donde la primera etiqueta marca el inicio del elemento y la segunda, precedida de una barra diagonal, marca el final.
  • HTML define la estructura de la página, organizando el contenido en encabezados, párrafos, listas, tablas, etc.

CSS (siglas de Cascading Style Sheets, 'hojas de estilo en cascada') es un lenguaje que gestiona el diseño y la presentación de las páginas web, complementando el HTML, que se encarga del contenido. Se llama "en cascada" porque permite usar múltiples hojas de estilo que heredan propiedades unas de otras.

  • CSS permite personalizar la apariencia de un sitio, definiendo reglas para elementos de estilo como fuentes, colores y tamaños, separándolos del contenido y mejorando la experiencia visual de los usuarios.
  • Las reglas CSS están compuestas por selectores y declaraciones. Un selector especifica qué elementos HTML se verán afectados, y las declaraciones definen los estilos que se aplicarán.

Ejemplo básico de HTML y CSS: 
<!DOCTYPE html>
<html> 
<head>
<title>
Ejemplo de HTML y CSS</title>
 <style>
      h1 {
        color: blue;
         text-align: center;
        }
        p {
        font-size: 16px;
        color: gray;
        }
</style>
</head>
    <body>
<h1>¡Hola, mundo!</h1>

<p>Este es un párrafo de ejemplo
que utiliza estilos definidos en CSS.
</p>
</body>
</html>
En este ejemplo, el HTML define la estructura de la página con un encabezado y un párrafo, mientras que el CSS dentro de la etiqueta <style> define que el encabezado será de color azul y centrado, y el párrafo será de color gris y tendrá un tamaño de fuente de 16px.

JavaScript 🧩

Conceptos básicos de JavaScript

Ver más

JavaScript es un lenguaje de programación que permite implementar funciones complejas en las páginas web.


<script> Se usa para incluir código JavaScript en una página HTML.
Puede ir en la sección <head> o al final del <body>. Es recomendable colocarlo al final del <body> para que el contenido HTML se cargue primero y mejorar el rendimiento de la página.
<script> No es una "etiqueta" de JavaScript en sí, sino una etiqueta de HTML que permite ejecutar código JavaScript.

Este lenguaje no usa etiquetas como en HTML, sino que trabaja con funciones, métodos y objetos para interactuar con el HTML y manipular el contenido de las páginas web.

Características de JavaScript:
  • Es un lenguaje de programación de alto nivel, interpretado y orientado a objetos.
  • Permite crear interactividad en las páginas web, como responder a eventos del usuario (clics, desplazamientos, pulsaciones de teclas), validar formularios, animar elementos, etc.
  • Es ejecutado por los navegadores web, por lo que no necesita ser compilado.
Interacción con HTML:
  • JavaScript puede manipular el DOM (Document Object Model) para modificar, eliminar o agregar elementos, estilos o contenido en la página web.
  • A través de funciones y métodos, se puede hacer que las páginas sean interactivas y dinámicas.
Ejemplo básico: 
<!DOCTYPE html>
<html>
<head>
<title>Ejemplo de JavaScript</title>
</head>
<body>
 <h1 id="titulo">¡Hola, mundo!</h1>
<button onclick="cambiarTexto()">Haz
 clic aquí</button>
        
<script>
function cambiarTexto() {
document.getElementById("titulo")
.innerText = "¡Has hecho clic!";
}
</script>
</body>
</html>

En este ejemplo, al hacer clic en el botón, la función cambiarTexto() cambia el texto del elemento con el id "titulo".

Etiquetas HTML 🤓☝️

Algunas etiquetas para tener en cuenta

Ver etiquetas
  • En HTML, los elementos se definen usando etiquetas. Las etiquetas son las construcciones básicas de HTML y se usan para marcar el inicio y el final de un elemento.
    Estas etiquetas vienen en pares: una etiqueta de apertura y una etiqueta de cierre.
  • <html> </html> Principio y final del documento HTML.
  • <head> </head> Contiene metadatos sobre el documento, como el título y enlaces a hojas de estilo o scripts.
  • <body> </body> El contenido visible de la página, donde se coloca todo lo que el usuario verá en la web.

    • Etiquetas semánticas dentro del <body>:
    • <header> </header> Define la cabecera de una página o sección, que generalmente contiene el logotipo, el menú de navegación o el título principal.
    • <nav> </nav> Define una sección de navegación que contiene enlaces a otras partes de la página o a diferentes páginas del sitio.
    • <section> </section> Define una sección genérica de contenido que puede contener encabezados, párrafos, imágenes, etc. Es útil para agrupar contenido relacionado.
    • <article> </article> Representa contenido independiente y auto-contenido, como un artículo de blog, una noticia o una entrada de foro.
    • <aside> </aside> Contenido que está relacionado con el contenido principal pero que es independiente, como barras laterales, citas o anuncios.
    • <footer> </footer> Define el pie de página de una página o sección, que suele contener información de contacto, derechos de autor o enlaces importantes.
    • <h1> al <h6> Definen los títulos y subtítulos de la página, siendo <h1> el más importante y <h6> el menos importante.
    • <p> </p> Define un párrafo de texto.
    • <img src=""> Añade una imagen a la página. El atributo src especifica la ruta de la imagen.
    • <a href=""> </a> Crea un hipervínculo. El atributo href define la URL a la que se enlaza.
    • <span> </span> Utilizado para aplicar estilos o manipular un fragmento específico de texto dentro de una línea.
    • <div> </div> Etiqueta genérica para agrupar elementos. No tiene un significado específico pero es útil para estructurar contenido y aplicar estilos.

Componentes de una regla CSS 🧐🔍

Selector, valor, propiedad etc.

Ver ejercicios

  • CSS no usa etiquetas como HTML, sino que usa selectores, propiedades y valores para aplicar estilos a los elementos HTML.

  • color: Define el color del texto.
  • background-color: Define el color de fondo de un elemento.
  • padding: Define el espacio interior entre el borde de un elemento y su contenido.
  • margin: Establece los márgenes de un elemento.
  • display: Controla el modelo de caja del elemento, como block, inline, flex, grid, etc.
  • text-align: Define la alineación del texto dentro de un elemento.
  • width: Define el ancho de un elemento.
  • height: Define el alto de un elemento.
  • border: Define el estilo, ancho y color del borde de un elemento (por ejemplo, border: 1px solid black;).
  • font-size: Define el tamaño de la fuente del texto.
  • font-family: Define la fuente del texto, como Arial, Verdana, "Times New Roman", etc.
  • background-image: Define una imagen de fondo para un elemento.
  • opacity: Define la transparencia de un elemento, donde 0 es completamente transparente y 1 es opaco.
  • box-shadow: Añade una sombra alrededor de un elemento (por ejemplo, box-shadow: 5px 5px 10px gray;).
  • text-transform: Controla la capitalización del texto, como uppercase (mayúsculas), lowercase (minúsculas), capitalize (primera letra en mayúscula).
  • position: Define cómo se posiciona un elemento en la página (por ejemplo, static, relative, absolute, fixed, sticky).
  • overflow: Controla qué ocurre si el contenido de un elemento es más grande que el área que lo contiene (valores: visible, hidden, scroll, auto).
  • z-index: Controla la superposición de elementos posicionados (un valor más alto lo coloca por encima de otros elementos con un valor más bajo).
  • transition: Permite crear animaciones suaves al cambiar de un estado a otro (por ejemplo, transition: all 0.3s ease;).

  • En un CSS cada componente tiene su nombre:



    .h1: selector
    color: blue;: declaración
    color: propiedad
    blue o #0000FF: valor
    { }: llaves

    Dentro de un CSS lo que vamos a ver es una regla completa: Incluye tanto selector como declaraciones dentro de llaves, todo en un solo bloque.


    (ejemplo: .h1 { color: blue; })

Node.js 🟢

Conceptos básicos de Node.js

Ver más

Node.js es un entorno de ejecución para JavaScript en el servidor. Permite usar JavaScript fuera del navegador para desarrollar aplicaciones del lado del servidor.


Node.js No es un lenguaje de programación en sí, sino una plataforma construida sobre el motor V8 de Google Chrome.
npm (Node Package Manager) es el gestor de paquetes para Node.js, utilizado para instalar y gestionar dependencias en los proyectos.
node Es el comando utilizado para ejecutar archivos JavaScript en el entorno de Node.js desde la línea de comandos.

Node.js y npm proporcionan varios comandos útiles para gestionar proyectos y ejecutar scripts.

  • node archivo.js - Ejecuta un archivo JavaScript en el entorno de Node.js.
  • npm init - Inicializa un nuevo proyecto Node.js creando un archivo package.json.
  • npm install - Instala un paquete y sus dependencias desde el registro de npm.
  • npm update - Actualiza todos los paquetes instalados a sus últimas versiones.
  • npm uninstall paquete - Desinstala un paquete del proyecto.
  • npm run script - Ejecuta un script definido en el archivo package.json.
  • npx comando - Ejecuta un comando de un paquete sin necesidad de instalarlo globalmente.

Git 📥

Conceptos básicos y comandos comunes de Git

Ver más

Git es un sistema de control de versiones distribuido que permite gestionar y rastrear cambios en el código fuente durante el desarrollo de software.

Git es ampliamente utilizado en la industria y facilita la colaboración en equipo, permitiendo a varios desarrolladores trabajar en un proyecto simultáneamente. Para más información, puedes visitar la documentación oficial de Git.

A continuación, algunos comandos básicos de Git:

  • cd /c: Este comando es útil cuando necesitas navegar a la raíz de la unidad C.
  • cd [nombre de carpeta]: Este comando permite navegar y moverte hacia la carpeta que especifiques.
  • mkdir [nombre de carpeta]: Crea una nueva carpeta con el nombre especificado.
  • ls : Muestra una lista de los archivos y carpetas en el directorio actual.
  • ls -a : Muestra todos los archivos y carpetas en el directorio actual, incluyendo archivos ocultos (.git).
  • git init: Inicializa un nuevo repositorio de Git en un proyecto. Esto convierte la carpeta actual en un repositorio de Git. Este comando crea una subcarpeta oculta llamada .git, donde Git almacena toda la información necesaria para gestionar el repositorio, como el historial de cambios y las configuraciones.
  • git remote add origin [URL]: Sirve para asociar el repositorio local con el repositorio remoto. Es decir, se usa para linkear el repositorio local con el remoto.
  • git clone [URL]: Clona un repositorio remoto en tu máquina local.
  • git add [archivo]: Añade archivos al área de preparación (staging area) para el próximo commit.
  • git add . : Añade todos los archivos y cambios al área de preparación (staging area), listo para ser comiteado.
  • git commit -m "mensaje": Realiza un commit con los archivos en el área de preparación y añade un mensaje descriptivo.
  • git push origin [rama] Envía los cambios locales a la rama especificada en el repositorio remoto.
  • git status: Muestra el estado actual del repositorio, incluyendo cambios no confirmados y archivos no rastreados.
  • git pull: Obtiene los últimos cambios del repositorio remoto y los fusiona con la rama local.
  • git push: Envía los commits locales al repositorio remoto.
  • git branch: Muestra una lista de ramas en el repositorio y resalta la rama actual.
  • git branch [rama]: Este comando solo crea la rama, pero no te cambia a ella.
  • git checkout [rama]: Cambia a una rama específica en el repositorio.
  • git checkout -b [rama]: Se usa para crear una nueva rama y cambiarse automáticamente a esa rama recién creada en un solo paso.
  • git branch -d [rama]: Borra una rama local con el nombre especificado, siempre que no tenga cambios pendientes.
  • git config --global user.name [Mi nombre]: Configura el nombre de usuario en Git. Este nombre aparecerá en los commits que realices.
  • git config --global user.email [mi_mail@mail.com]: Configura el correo electrónico asociado con el usuario en Git. También aparece en los commits.
  • git log: Muestra el historial de commits en el repositorio, con detalles de cada commit, como el autor, fecha y mensaje de commit.
  • git merge [rama]: Se utiliza para combinar los cambios de una rama especificada con la rama en la que estás actualmente.

GitHub 🗃️

Uso básico de GitHub y comandos comunes

Ver más

GitHub es una plataforma de alojamiento para proyectos de desarrollo de software que utiliza Git, un sistema de control de versiones distribuido. GitHub permite a los desarrolladores colaborar en proyectos de código, gestionar versiones de software, y compartir su trabajo con otros, tanto en proyectos públicos como privados.

A continuación, algunos comandos y conceptos básicos para trabajar con GitHub:

  • git remote add origin

    Vincula tu repositorio local con un repositorio remoto en GitHub
    Este comando asocia tu repositorio local con una URL de repositorio remoto, permitiendo sincronización entre ambos.

  • git push origin main

    Envía tu rama principal (main) al repositorio remoto en GitHub por primera vez
    Este comando sube tus cambios a la rama principal del repositorio remoto y configura `main` como rama predeterminada para futuros `push`.

  • Pull Request

    Una solicitud de extracción permite proponer cambios en el repositorio de otra persona
    Los PR son cruciales para colaborar, revisando cambios y discutiendo mejoras antes de fusionarlos al proyecto principal.

  • Git clone

    Clona un repositorio de GitHub en tu máquina local para trabajar en él
    Clonar un repositorio descarga una copia completa de ese repositorio en tu máquina local, incluyendo todo su historial de cambios.

  • git pull origin main

    Actualiza tu repositorio local con los cambios más recientes del repositorio remoto en la rama principal
    Este comando sincroniza tu copia local del repositorio con los cambios más recientes realizados en la rama principal del repositorio remoto.

  • Git push

    Envía tus commits locales al repositorio remoto en GitHub
    El comando `push` actualiza el repositorio remoto con los commits hechos en tu repositorio local, compartiendo así los cambios con otros colaboradores.

  • Acciones de github

    Son flujos de trabajo automáticos que puedes configurar para CI/CD, pruebas, despliegue, etc.
    Las GitHub Actions te permiten automatizar tareas comunes de desarrollo, como ejecutar pruebas automáticas cada vez que se hace un `push` a una rama.

Comparación: Netlify vs GitHub Pages 🌐

Dos populares plataformas para alojar sitios web estáticos

Ver más

Netlify y GitHub Pages son dos servicios ampliamente utilizados para alojar sitios web estáticos. A continuación, te presentamos una comparación de sus características principales:

  • Facilidad de Uso:
    Netlify: Ofrece una interfaz intuitiva y una configuración fácil de usar, incluso para principiantes. Permite implementar directamente desde un repositorio de GitHub, GitLab o Bitbucket.
    GitHub Pages: Requiere que el repositorio esté configurado correctamente, pero la implementación es automática si el proyecto está en GitHub. No tiene una interfaz tan visual como Netlify.

  • Configuración del Dominio Personalizado:
    Netlify: Ofrece una configuración muy sencilla para agregar un dominio personalizado y proporciona un certificado SSL gratuito.
    GitHub Pages: También permite usar un dominio personalizado, pero requiere un poco más de configuración manual.

  • Velocidad y Rendimiento:
    Ambos servicios ofrecen un rendimiento excelente para sitios estáticos, con Netlify ofreciendo optimizaciones adicionales y CDN de forma nativa.

  • Integraciones y Funcionalidades Adicionales:
    Netlify: Tiene integraciones para formularios, autenticación, y funciones serverless, lo que permite agregar funcionalidades adicionales a sitios estáticos.
    GitHub Pages: Está más limitado a la funcionalidad básica de alojar contenido estático.
